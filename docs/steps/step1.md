# Step 1: Open the PR

## What to Do

1. Click on `pull_request_url` to open the Pull Request on GitHub.
2. Read the **PR title** to understand what the change is about.
3. Read the **PR description** (if available) to understand:
   - What repository/project this is
   - What the PR is changing and why
   - Any additional context the author has provided

## Goal of This Step

Get the high-level context about the PR. Knowing what the change is about allows you to better evaluate whether a review comment is relevant, correct, or out of place. Without this context, you might misinterpret the purpose of the comment.
