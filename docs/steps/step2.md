# Step 2: Go to the Discussion

## What to Do

1. Click on `discussion_url` — this takes you directly to the review comment on GitHub.
2. You will see the comment displayed alongside its surrounding diff hunk (the changed lines of code around the comment).
3. Read the comment in its visual context within the diff.
4. **Mandatory checkpoint:** Verify that the comment shown on GitHub matches exactly the `body` field in the input data. If it does not match, flag the task.

## Goal of This Step

See the comment in its real context within the PR. The discussion view shows you the comment alongside the code it refers to, which is essential for understanding what the reviewer is pointing out.
