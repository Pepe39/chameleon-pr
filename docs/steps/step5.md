# Step 5: Re-read and Deeply Understand the Comment

## What to Do

1. Go back to the `body` field in the input data.
2. Now that you have full context (PR, diff, and possibly the repo), re-read the comment carefully.
3. Ask yourself these questions:
   - **What specific issue is this comment pointing out?**
   - **Is the comment's claim factually correct?** Verify it against the actual code.
   - **What context would a reviewer need to make this comment?** Only the diff? The full file? Other files?
   - **How severe is the issue itself?** Not the comment's tone, but the actual impact of the problem.
4. If anything is still unclear, **repeat Steps 2–4** as many times as needed. Navigate between the diff, discussion, and repo until you fully understand.

## Goal of This Step

Ensure you fully understand the comment before labeling it. Re-reading the comment after understanding the full diff often changes your initial impression. Do not rush to label — accuracy matters more than speed.
